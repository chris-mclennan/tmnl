## Two tracks

tmnl runs in two modes from the same window. **Shell mode** is your daily driver — a GPU terminal hosting your `$SHELL` with mouse, OSC 133, and offline AI completion. **Native mode** is the differentiator — apps connect over a Unix socket and ship typed cell `Frame`s straight into tmnl's grid, skipping the ANSI round-trip entirely.

## Why "native mode" matters

If you've ever tried to render a real UI inside a terminal — fine-grained graphics, knobs, structured grids — you've fought ANSI. tmnl's native mode is the escape: an app opens a Unix socket, ships `Frame` messages of typed cells, and tmnl draws them on the GPU. No `vt100` parser between the app and the screen. App and terminal agree on a `Grid` instead of trading bytes.

[`mnml`](https://mnml.sh) (a NvChad-style terminal IDE) and [`mixr`](https://mixr.sh) (a terminal DJ app) both ship a `--blit <socket>` mode that connects them as native tmnl tabs. Inside tmnl, mixr's crossfader renders as actual graphics, not unicode block art. mnml gets native-tab integration with workspace switching.

You don't have to use any of this. tmnl works fine as a regular terminal emulator hosting your shell. The cell-streaming layer just makes it the host for a class of apps that couldn't exist before. See [Native tabs](/manual/native-tabs/) for the full protocol walkthrough.

## State of the project

tmnl is `v0.0.x`. It's solid as a daily driver on macOS for me (the author), but it has sharp edges:

- **macOS-only today.** The wgpu pipeline is cross-platform; the menu bar, pty hosting, and SCM_RIGHTS handoff are not yet ported off macOS. Windows + Linux are roadmap.
- **No scrollback search, no splits, no font config UI.** Single-window, single-pane-per-tab, single font baked in. Splits and search are the next big tracks.
- **Native-mode SDK is unpublished.** The protocol is stable enough that mnml + mixr ride it daily, but the published crate / capability negotiation / second-reference-client work is still ahead.

For shipped features see [FEATURES.md](https://github.com/chris-mclennan/tmnl/blob/master/FEATURES.md); for the roadmap see [`.local/PLAN.md`](https://github.com/chris-mclennan/tmnl/blob/master/.local/PLAN.md) (or just file an issue).

## The family

tmnl is one of three small Rust tools by [Chris McLennan](https://github.com/chris-mclennan). They share an aesthetic (`> name` prompt-style wordmark, charcoal + warm-orange palette) and a protocol — the [`tmnl-protocol`](https://github.com/chris-mclennan/tmnl-protocol) wire format lets the other two run as native tmnl tabs.

- **tmnl** — this terminal
- **[mnml](https://mnml.sh)** — a NvChad-style terminal IDE (vim or standard editing, LSP, git, fuzzy everything, baked-in HTTP client)
- **[mixr](https://mixr.sh)** — a terminal DJ app (Beatport streaming, Claude DJ, MIDI controllers)

See [the family page](/family/) for the side-by-side.
