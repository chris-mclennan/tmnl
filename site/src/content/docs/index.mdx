## What is tmnl

tmnl is a native terminal emulator written in Rust, using `wgpu` and `winit`. It runs on macOS, Linux, and Windows.

Beyond rendering regular shell sessions, tmnl exposes a **structured-cell display surface** — apps can connect over a Unix socket (or named pipe on Windows) and stream their UI as a grid of cells, rather than ANSI-encoded text. This is the protocol `mnml` and `mixr` use to render as native tabs.

## The family

tmnl is one of three small Rust tools that work together:

- **[mnml](https://mnml.sh)** — a NvChad-style terminal IDE (vim or standard editing, LSP, git, fuzzy everything, baked-in HTTP client)
- **[mixr](https://mixr.sh)** — a music app
- **tmnl** — the host that lets mnml and mixr render as native tabs

You don't need all three. tmnl works fine as just a terminal emulator. But if you install mnml or mixr alongside, tmnl becomes the launcher and they appear as siblings to your shell tabs.
